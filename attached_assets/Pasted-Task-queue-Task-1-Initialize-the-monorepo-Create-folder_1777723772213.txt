Task queue
Task 1 — Initialize the monorepo Create folder structure: /site (Jekyll), /worker (Cloudflare Worker), /shared(types/constants), root README.md. Install Ruby + Jekyll in the Repl, install Node + wrangler in /worker. Add .gitignore, .env.example. Done when: cd site && bundle exec jekyll serve and cd worker && npx wrangler dev both run without errors on a hello-world.
Task 2 — Jekyll landing page Build the marketing page in /site: hero with the Algosize positioning line, three feature blocks (cost / security / algorithms), pricing block with one Stripe-linked CTA button, footer. No JavaScript frameworks — vanilla CSS, mobile-responsive. The CTA button hits /api/checkout (will return 404 until task 5). Done when: page renders, looks intentional (not a Bootstrap default), CTA button exists with correct href.
Task 3 — Worker scaffolding + routing In /worker, set up wrangler.toml with KV namespace bindings (SESSIONS, USERS), env vars for Stripe keys and JWT secret. Create a router (itty-router or hono) with stub routes: POST /api/checkout, POST /api/stripe/webhook, GET /api/me, POST /api/analyze/cost, POST /api/analyze/vuln, POST /api/analyze/algo. All return 501 for now. Done when: curl localhost:8787/api/me returns 501 with the right CORS headers.
Task 4 — JWT auth utilities In /worker/src/auth.js, implement issueJWT(userId, email, subStatus) and verifyJWT(token) using Web Crypto (HS256). Store active sessions in KV with 30-day TTL. Add a requireAuthmiddleware that reads cookie or Bearer token. Done when: unit test (or curl flow) issues a token, verifies it, and rejects a tampered one.
Task 5 — Stripe checkout endpoint Implement POST /api/checkout: creates a Stripe Checkout Session for the monthly plan, returns the redirect URL. Use Stripe's REST API directly via fetch — no Node SDK (Workers don't run it cleanly). Hardcode the price ID via env var. Done when: clicking the landing page CTA redirects to a real Stripe checkout page in test mode.
Task 6 — Stripe webhook handler Implement POST /api/stripe/webhook: verify signature using Web Crypto, handle checkout.session.completed and customer.subscription.deleted. On success, create user record in USERS KV ({email, stripeCustomerId, subStatus, createdAt}), issue JWT, set HttpOnly Secure cookie, redirect to /dashboard. Done when: completing a test-mode checkout creates a KV record and lands on /dashboard with a valid session cookie. Test webhook signature verification with a tampered payload — must reject.
Task 7 — Cost analyzer endpoint Implement POST /api/analyze/cost behind requireAuth. Input: JSON describing cloud usage (services, monthly spend, regions). Output: {currentSpend, suggestions: [{title, impact, savingsEstimate}], totalSavingsPct}. For MVP: rule-based engine that looks for common waste patterns (idle resources, oversized instances, egress, unreserved compute). No LLM call yet — leave a clean interface to swap one in later. Done when: posting a sample payload returns three real-feeling suggestions. Important: the savings number must be defensible. Don't return "40%" by default — return what the rules actually find, even if it's 6%.
Task 8 — Vulnerability scanner endpoint Implement POST /api/analyze/vuln behind requireAuth. Input: pasted code (string) or list of files. Output: {findings: [{severity, type, line, snippet, recommendation}]}. Detect: hardcoded secrets (regex for AWS keys, GitHub tokens, generic API key patterns), eval() / exec(), SQL string concatenation, unparameterized template strings in queries, http:// URLs in production-looking config. Mark severity honestly. Done when: a sample file with three planted issues returns three findings, no false positives on a clean file.
Task 9 — Algorithm optimizer endpoint Implement POST /api/analyze/algo. Input: function source. Output: {currentComplexity, suggestions, optimizedExample}. For MVP: detect nested loops on the same array (O(n²) → suggest hash map), repeated array operations inside loops, recursive calls without memoization. Be modest — return "we couldn't analyze this" rather than fake suggestions when patterns don't match. Done when: a planted O(n²) function returns the right diagnosis; a clean function returns "no obvious improvements found."
Task 10 — Dashboard page In /site, build /dashboard/index.html (Jekyll layout) with three sections: cost analysis, vulnerabilities, algorithms. Each has an input area (JSON editor or code paste) and a results panel. Use plain JS (fetch) to call the Worker endpoints with the cookie. Include a logout button that calls /api/logout. Add a simple chart for cost savings — use a tiny library like Chart.js via CDN, not a framework. Done when: logged-in user can paste sample data into all three modules and see real responses; logged-out user gets redirected.
Task 11 — End-to-end smoke test Write a manual test script (markdown checklist) covering: visit landing → click CTA → Stripe test checkout → land on dashboard → run all three analyzers → logout → verify dashboard now blocks. Fix anything broken. Done when: the checklist passes start to finish in one sitting.
Task 12 — Deployment handoff doc Generate DEPLOY.md with exact commands for: pushing /site to GitHub Pages with custom domain, deploying /worker via wrangler deploy, setting Cloudflare secrets (wrangler secret put STRIPE_SECRET_KEY, etc.), pointing algosize.com DNS, configuring Stripe webhook URL to the deployed Worker, swapping Stripe test keys for live. Don't execute these from inside Replit — you'll run them locally. Done when: doc is complete enough that you can deploy without re-asking the agent.
