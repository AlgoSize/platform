
A modal appears with Client ID and Client Secret. Copy both — you'll paste them in Step 2. (You can re-open them anytime from the Credentials list.)
Step 2 — Set the secrets on the production Cloudflare Worker
Open your terminal (locally, where you have the repo cloned and wrangler authenticated via wrangler login):

cd worker
wrangler secret put GOOGLE_CLIENT_ID --env production
# Paste the Client ID when prompted, hit Enter.
wrangler secret put GOOGLE_CLIENT_SECRET --env production
# Paste the Client Secret when prompted, hit Enter.

Verify both are stored:

wrangler secret list --env production

You should see GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET in the output (values are not shown, only names — that's normal).

Step 3 — Deploy the Worker
cd worker
wrangler deploy --env production

You should see output ending with something like:

Uploaded algosize (X.XX sec)
Published algosize (X.XX sec)
  https://algosize.guillaumelauzier.workers.dev

Step 4 — Smoke-test the OAuth flow
A. Confirm /start redirects to Google:

curl -sI https://algosize.guillaumelauzier.workers.dev/api/auth/google/start

Expected: HTTP/2 302 with a location: header pointing to https://accounts.google.com/o/oauth2/v2/auth?.... If you see location: /?auth=google_not_configured, the secrets aren't set — go back to Step 2.

B. End-to-end browser test:

Open https://algosize.guillaumelauzier.workers.dev/api/auth/google/start in an incognito window.
Pick a Google account → grant consent.
You should land on https://algosize.guillaumelauzier.workers.dev/dashboard/ (this will 404 today since the static dashboard isn't served by the Worker — see "Known limitation" below).
Confirm the session cookie was set:
curl -b "algosize_session=PASTE_COOKIE_FROM_DEVTOOLS" \
  https://algosize.guillaumelauzier.workers.dev/api/me

Expected: 200 with { "email": "...", "plan": "free", ... }.
Step 5 — Verify magic-link flow uses the same URL
The magic-link flow now also uses https://algosize.guillaumelauzier.workers.dev for verify links and post-verify redirects (since both auth flows read SITE_ORIGIN).

For magic-link emails to actually be delivered, you also need:

wrangler secret put GOOGLE_SERVICE_ACCOUNT_JSON --env production
# Paste the full service account JSON file contents.

Without it, POST /api/auth/request-link returns 200 "check your inbox" but no email is sent (this is intentional — fail-soft).
